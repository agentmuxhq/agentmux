## Open PRs

### PR #241 — open, needs review + merge
**fix: resync existing terminals after backend restart**
Branch: `agenta/fix-term-resync-after-backend-restart`

**Bug:** After sidecar crash + restart, existing terminal panes accept no input. New terminals work fine.

**Root cause:** `TermResyncHandler` in `term.tsx` only resyncs on `connStatus` change (SSH/WSL connection). Local terminals stay `status: "connected"` throughout a backend crash cycle — the effect guard `isConnected == wasConnected` blocks the resync. The PTY is dead on the new backend but `ControllerResyncCommand` is never called.

**Fix:** Added second `createEffect` watching `backendStatusAtom`. When it transitions to `"running"` from non-running, calls `resyncController("backend-restart")` on every mounted terminal.

File changed: `frontend/app/view/term/term.tsx` (+15 lines)
