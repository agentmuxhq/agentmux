## Current State — What Doesn't Work

1. **`task cef:dev` broken on Windows** — PowerShell `$` variable escaping in `cef:bundle:windows` Taskfile task. `$$` produces literal `$$` in PowerShell instead of `$`. Workaround: run Vite + CEF host manually (see Build Commands below).

2. **Hostname popover not in Tauri build** — `get_host_info` IPC command only exists in CEF host, not in `src-tauri/src/commands/platform.rs`. Popover shows only hostname when running under Tauri.

3. **Multi-window** — commands exist but not wired to UI
4. **Cross-window drag** — 10 commands stubbed
5. **File drop into terminal** — needs CefDragHandler
6. **React cleanup** — dead deps in package.json (`react`, `react-dom`, `@types/react`), some `React.*` types still in code

## Build Commands

### CEF Portable Build
```bash
export PATH="/c/Program Files/Microsoft Visual Studio/18/Community/Common7/IDE/CommonExtensions/Microsoft/CMake/Ninja:$PATH"
bump patch -m "description" --commit
cargo build --release -p agentmux-cef -p agentmux-launcher
task cef:bundle   # may fail — see workaround below
npx vite build --mode production --config vite.config.tauri.ts
task cef:package:portable
```

### CEF Dev Mode (workaround for broken `task cef:dev`)
```bash
# Terminal 1: start Vite
npx vite --config vite.config.tauri.ts --port 5173

# Terminal 2: build and run CEF
export PATH="..../CMake/Ninja:$PATH"
cargo build --release -p agentmux-cef
cp target/release/agentmux-cef.exe dist/cef/
cd dist/cef && ./agentmux-cef.exe --url=http://localhost:5173
```

### Tauri Dev Mode
```bash
task dev
```
