## What Is This

AgentMux has two host/shell variants:
- **Tauri build** — uses system WebView2 (Edge/Chromium). Small (30 MB), depends on OS runtime.
- **CEF build** — bundles its own Chromium (CEF 146). Large (311 MB) but faster startup, lower memory, identical rendering across machines.

Same frontend (SolidJS + xterm.js), same Rust backend sidecar. Only the host differs.

## Current State — What Works

- Window launches, resizes, minimizes, maximizes, drags (frameless with custom title bar)
- Backend sidecar spawns via Job Object (auto-killed on host exit)
- IPC bridge: frontend calls ~40 commands via HTTP POST to embedded axum server
- Terminal panes render, accept input, handle SIGINT
- Sysinfo, agent, forge, identity, help panes
- Context menus (JS overlay, not native)
- Pane DnD (split, move) and tab DnD (reorder) via pragmatic-dnd
- DevTools via remote debugging on port 9222
- Window transparency/opacity (DWM + WS_EX_LAYERED)

## Current State — What Doesn't Work

1. **Multi-window** — commands exist (`open_window_at_position`, `list_windows`, `focus_window`) but aren't wired to frontend triggers. No "new window" button or Ctrl+Shift+N.

2. **Cross-window drag** — 10 drag commands stubbed (`start_cross_drag`, `accept_cross_drag`, etc.) but not implemented. Needed for pane/tab tear-off between windows.

3. **File drop into terminal** — dropping files shows no path. Needs `CefDragHandler::OnDragEnter` to extract file paths and inject as terminal text.

4. **React cleanup** — `react`, `react-dom`, `@types/react` are dead dependencies in `package.json`. Some code still uses `React.KeyboardEvent` and `React.RefObject` types (in `frontend/util/keyutil.ts` and `frontend/layout/lib/nodeRefMap.ts`). Needs migration to native DOM types.

5. **Version isolation broken** (HIGH) — CEF uses shared `ai.agentmux.cef/` root. Settings bleed between versions. Tauri uses per-version directories (`ai.agentmux.app.v0-31-NNN/`). CEF should use `ai.agentmux.cef.v0-32-NNN/`. Also `get_data_dir`/`get_config_dir` return the unversioned root to the frontend. Full analysis in `specs/cef-isolation-audit.md`.

```
agentmux-cef-0.32.111-x64-portable/
  agentmux.exe       325 KB   ← launcher (SetDllDirectoryW → spawns runtime/agentmux-cef.exe)
  README.txt
  runtime/
    agentmux-cef.exe           real CEF host
    agentmuxsrv-rs.x64.exe    backend sidecar
    libcef.dll                 251 MB CEF core
    libEGL.dll, libGLESv2.dll, d3dcompiler_47.dll   GPU (ANGLE)
    chrome_elf.dll, icudtl.dat, v8_context_snapshot.bin
    *.pak, locales/en-US.pak
    frontend/                  built SolidJS app
```

**Why a launcher?** `libcef.dll` is a load-time dependency — OS loader needs it before `main()`. The launcher calls `SetDllDirectoryW("runtime/")` then spawns the real binary.

**File stripping applied:** SwiftShader removed (deprecated upstream), WebGPU DLLs removed, 49 non-English locales removed. GPU support preserved.

## Key Architecture Decisions

- **Launcher crate** (`agentmux-launcher/`) — separate from `agentmux-cef` because the CEF crate links `libcef.dll` at load time. Can't be in the same binary.
- **CEF Views mode** (not native window) — cross-platform window management via CEF's Views framework. Frameless via `is_frameless() = 1`.
- **IPC via HTTP** — frontend uses `fetch("http://127.0.0.1:{port}/ipc")` instead of Tauri's `invoke()`. Detected at startup via `window.__AGENTMUX_IPC_PORT`.
- **Alloy runtime style** — hardcoded, no CLI flag needed. Alloy = CEF's single-process-model-friendly style.
- **Settings shared across versions** — intentional, matches Tauri behavior.
