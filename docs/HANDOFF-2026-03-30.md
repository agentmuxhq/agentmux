## Architecture

```
agentmux.exe (325 KB launcher)
  └── runtime/agentmux-cef.exe (CEF host)
        ├── runtime/agentmuxsrv-rs.x64.exe (backend sidecar)
        ├── runtime/wsh.exe (shell integration)
        ├── runtime/frontend/ (SolidJS app)
        └── runtime/libcef.dll + CEF runtime files
```

Two host variants from same codebase:
- `task dev` — CEF dev mode (Vite + CEF host)
- `task tauri:dev` — Tauri dev mode (Vite + WebView2)

## What Works

- Window: launch, resize, minimize, maximize, drag (frameless CEF Views)
- Multi-window: click version button, instance counter (1), (2), (3)...
- Backend sidecar: Job Object lifecycle, version-isolated data dirs
- IPC: HTTP POST bridge, auth token, ~40 commands
- All pane types: terminal, sysinfo, agent, forge, identity, help
- Context menus, pane/tab DnD, per-pane zoom
- DevTools on port 9222
- Hostname popover with host info + LAN discovery toggle
- Shell integration (wsh)

## What Doesn't Work

1. **Secondary window edge resize** — frameless WS_POPUP has no resize borders. Users resize via maximize/restore. Fix requires WM_NCHITTEST subclassing or moving window creation to CEF UI thread.

2. **`get_host_info` not in Tauri build** — hostname popover shows only hostname when running under Tauri. Need to add the command to `src-tauri/src/commands/platform.rs`.

3. **Cross-window drag** — 10 commands stubbed for pane/tab tear-off between windows.

4. **File drop into terminal** — needs CefDragHandler for file path extraction.

5. **React cleanup** — dead deps in package.json, some `React.*` types in frontend code.

6. **CEF build fragility** — `cef-dll-sys` cmake cache sometimes corrupts (missing `libcef_dll/` after partial extraction). Fix: `rm -rf target/release/build/cef-dll-sys-*` and rebuild. May need manual copy of `libcef_dll/` from extracted tarball if access denied persists.

## Build Commands

```bash
# Ninja must be on PATH
export PATH="/c/Program Files/Microsoft Visual Studio/18/Community/Common7/IDE/CommonExtensions/Microsoft/CMake/Ninja:$PATH"

# ALWAYS bump before portable builds
bump patch -m "description" --commit
bump verify

# Build
cargo build --release -p agentmux-cef -p agentmux-launcher
task cef:bundle
task build:wsh
npx vite build --mode production --config vite.config.tauri.ts
task cef:package:portable

# Dev mode
task dev  # launches CEF with Vite hot reload

# If cef-dll-sys build fails with missing libcef_dll:
rm -rf target/release/build/cef-dll-sys-*
# then rebuild
```

## CRITICAL: Build Integrity Rule

**NEVER ship a portable with a mismatched version.** If the build fails, fix it immediately. Do not reuse an old binary and name it with the new version. This caused a version mismatch earlier in this session (status bar showed 0.33.3 but filename said 0.33.4).
