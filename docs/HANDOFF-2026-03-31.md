### Direct-to-main fixes (0.33.5 → 0.33.6)

- **Dev workflow decoupling** — `cef:dev:serve` copies `dist/cef/` → `dist/cef-dev/` before launch. Running CEF never locks the staging dir, so rebuilds work without killing the process.
- **CEF_PATH** — stable extraction path in `.cargo/config.toml` prevents 155MB re-download on every version bump. Also sets `CMAKE_MAKE_PROGRAM` for Ninja.
- **Window icon** — loads app icon from embedded exe resource via `WM_SETICON`. Shows in taskbar and Alt+Tab.
- **Sidecar rebuild** — the sidecar binary was stale (cargo cache didn't detect the `network_lan_discovery` setting addition). Force-rebuilt with `touch` on source.

## What Works

- **CEF host** — frameless window, resize, minimize, maximize, drag
- **Multi-window** — click version button, each window gets instance number (1), (2), (3)
- **Backend sidecar** — Job Object lifecycle, version-isolated data dirs (`ai.agentmux.cef.v0-33-6/`)
- **IPC** — HTTP POST bridge with auth token, ~40 commands
- **All pane types** — terminal, sysinfo, agent, forge, identity, help
- **Context menus** — JS overlay
- **Pane/tab DnD** — split, move, reorder
- **DevTools** — port 9222
- **Hostname popover** — click hostname → host info, instance details, LAN discovery toggle, ports
- **mDNS opt-in** — disabled by default, no Windows Firewall prompt
- **wsh** — shell integration binary in `runtime/wsh.exe`
- **Window icon** — app icon in taskbar + Alt+Tab
- **Dev isolation** — `dist/cef-dev/` session copy, never locks staging dir
- **CEF_PATH** — fast builds, no SDK re-download

## What Doesn't Work

1. **Secondary window edge resize** — `WS_POPUP` has no resize borders. Users use maximize/restore. Fix requires `WM_NCHITTEST` subclassing or UI-thread window creation.

2. **`get_host_info` not in Tauri** — hostname popover shows only hostname under Tauri. Need command in `src-tauri/src/commands/platform.rs`.

3. **Cross-window drag** — 10 commands stubbed for pane/tab tear-off.

4. **File drop into terminal** — needs `CefDragHandler`.

5. **React cleanup** — dead deps (`react`, `react-dom`, `@types/react`), some `React.*` types.

6. **Hostname popover right-align CSS** — was lost in a squash merge, re-applied but may need verification. Check `.status-bar-popover.host-popover { left: auto; right: 0; }` in `StatusBar.scss`.

## Build Commands

```bash
# Dev mode (CEF + Vite hot reload)
task dev

# Portable build
bump patch -m "description" --commit
cargo build --release -p agentmux-cef -p agentmux-launcher
task cef:bundle
task build:backend
task build:wsh
npx vite build --mode production --config vite.config.tauri.ts
task cef:package:portable

# If cef-dll-sys fails (missing libcef_dll):
rm -rf target/release/build/cef-dll-sys-*
# then rebuild — CEF_PATH in .cargo/config.toml skips re-download

# ALWAYS verify binary versions match before shipping:
grep -ao "0\.33\.[0-9]" target/release/agentmux-cef.exe | sort -u
grep -ao "0\.33\.[0-9]" dist/bin/agentmuxsrv-rs.x64.exe | sort -u
```

## Critical Rules

1. **Never ship mismatched versions.** If build fails, fix it. Don't reuse old binaries.
2. **Always rebase feature branches on main before merging** — squash merges can lose code from other PRs.
3. **Verify sidecar version** — cargo may cache stale binaries. Use `touch` on source to force rebuild if needed.
