# AgentMux v0.14.0 Build Retro - 2026-01-09

## Summary

Attempted to build AgentMux v0.14.0 on gamerlove. Build ultimately not completed due to multiple infrastructure issues that required cleanup first.

## Issues Encountered

### 1. GOPATH Collision
- **Problem:** Wavemux repo was checked out directly to `C:\Users\asafe` (user home)
- **Impact:** Go's default GOPATH (`~/go`) created `C:\Users\asafe\go\pkg\mod` which Go then interpreted as `github.com/a5af/agentmux/go/pkg/mod` causing "import path should not have @version" errors
- **Fix:** Removed `go\pkg\mod` directory, but root cause was repo location

### 2. Frontend Build OOM
- **Problem:** Vite build ran out of memory (1GB allocation failed)
- **Cause:** Node.js default heap limit (~4GB) insufficient for large frontend build
- **Fix:** Set `NODE_OPTIONS=--max-old-space-size=8192`

### 3. Electron-Builder Deprecated Option
- **Problem:** `win.console` option deprecated in electron-builder v26.4.0
- **Fix:** AgentG removed the option from `electron-builder.config.cjs`

### 4. Stale wave.sock Files
- **Problem:** Old socket files blocking packaging
- **Fix:** Cleaned up before retry

### 5. Repo Location Chaos
- **Problem:** Repo files mixed with user profile folders in `C:\Users\asafe`
- **Impact:** Confusing, caused GOPATH issues, hard to manage
- **Resolution:** Cleaned up asafe folder, cloned fresh to `C:\Systems\agentmux-repo`

## Actions Taken

1. Cleaned all repo files from `C:\Users\asafe` (kept user profile folders)
2. Created `C:\Systems\agentmux-repo` with fresh clone
3. Checked out `agent2/v0.14.0-release` branch

## Next Steps

1. Run build from new location: `C:\Systems\agentmux-repo`
2. Set proper GOPATH (e.g., `C:\Go` or `C:\Users\asafe\go` but NOT inside project)
3. Use `NODE_OPTIONS=--max-old-space-size=8192` for frontend build
4. Copy successful build to Desktop

## Lessons Learned

- Never clone repos directly to user home directory
- GOPATH must not overlap with project directory
- Large Electron/Vite builds need increased Node heap
- Keep build infrastructure clean and well-documented
